## 💻 技術選定基準

### 採用する技術
- **シンプル**: 学習コストが低い
- **軽量**: 個人PCで動作
- **実績**: 枯れた技術
- **保守性**: メンテナンスしやすい

### 採用しない技術
- 複雑なマイクロサービス
- 過度な抽象化
- 最新すぎる技術（安定性不明）
- 重量級フレームワーク

---

## 📊 パフォーマンス目標

### 個人PC環境
- CPU: 4コア以上
- メモリ: 8GB以上
- ストレージ: SSD推奨

### 目標レスポンス
- バックテスト: < 10秒（100銘柄 x 1年）
- 市場スキャン: < 5秒
- UI操作: < 1秒
- データ取得: < 3秒

## 🔧 開発ガイドライン

### コーディング規約
1. **シンプルに書く** - 複雑なコードは避ける
2. **コメントを書く** - 将来の自分のために
3. **テストを書く** - 重要な機能のみ
4. **DRYを守る** - ただし過度な抽象化は避ける

### ファイル構成
```
src/
├── base.py           # 共通基底クラス
├── strategies/       # 取引戦略
├── risk/            # リスク管理
├── data/            # データ取得・処理
├── ui/              # UI コンポーネント
└── utils/           # ユーティリティ
```

### 命名規則
- **クラス**: PascalCase
- **関数**: snake_case
- **定数**: UPPER_SNAKE_CASE
- **プライベート**: _leading_underscore

## 🎨 UI/UX 方針

### デザイン原則
1. **シンプル** - 必要な情報だけ表示
2. **直感的** - 説明なしで使える
3. **高速** - 待たせない

- 詳細分析: 必要時のみ
- 設定: 最小限
